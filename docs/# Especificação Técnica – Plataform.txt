# Especificação Técnica – Plataforma Web CANOMAR

## 1. Objectivos Estratégicos

### 1.1 Presença Digital Institucional

Implementar uma plataforma web institucional para o **CANOMAR – Centro de Formação**, com o objectivo de:

* Posicionar o centro como referência em formação profissional
* Disponibilizar informação clara sobre cursos e serviços
* Reforçar credibilidade institucional
* Melhorar comunicação com alunos

### 1.2 Digitalização Administrativa

Automatizar processos actualmente manuais:

* Inscrições online
* Matrículas digitais
* Gestão documental
* Comunicação institucional
* Emissão de certificados

Resultado esperado:

* Redução de custos administrativos
* Menos erros humanos
* Processos mais rápidos

### 1.3 Aquisição de Novos Alunos

A plataforma deve funcionar como **canal comercial activo**, com:

* Catálogo optimizado
* Call-to-actions
* Inscrição simples
* Processo transparente

### 1.4 Conformidade Técnica

O sistema deve garantir:

* Design responsivo (mobile-first)
* Segurança de dados
* Encriptação SSL
* Conformidade GDPR ou equivalente
* Backups automáticos
* Monitorização

Integrações obrigatórias:

* Google Analytics
* Email marketing
* Google Maps
* Pagamentos locais

---

# 2. Arquitectura do Sistema

## 2.1 Estrutura Geral

O sistema será dividido em:

### Área Pública

Acesso sem autenticação.

Inclui:

* Página inicial
* Cursos
* Sobre nós
* Contactos
* Notícias
* Inscrição

---

### Área Privada

Acesso com autenticação.

Perfis:

* Aluno
* Professor
* Administrador

Sistema de permissões baseado em roles.

Exemplo:

| Função    | Acesso           |
| --------- | ---------------- |
| Visitante | Área pública     |
| Aluno     | Portal aluno (painel dedicado)     |
| Professor | Portal professor (painel dedicado) |
| Admin     | Sistema completo (painel dedicado) |

# 9. Stack Tecnológica Recomendada

## Backend

**Laravel 11 ou 12**

Responsável por:

* API interna
* Autenticação
* Permissões
* Processamento
* Uploads
* Certificados
* Emails

Vantagens:

* Segurança robusta
* ORM poderoso (Eloquent)
* Jobs e filas
* Facilidade de manutenção

---

## Painel Administrativo

**Filament**

Responsável por:

* Área administrativa
* Área professor
* Área aluno

Benefícios:

* CRUD rápido
* Dashboards
* Permissões
* Formulários dinâmicos
* Upload de documentos

Redução estimada:

**40–60% do tempo de desenvolvimento**

---

## Base de Dados

MySQL

Motivos:

* Simples manutenção
* Hosting barato
* Compatível com tudo
* Backups fáceis
